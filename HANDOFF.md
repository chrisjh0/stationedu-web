## Tech Stack

| Layer | Technology |
|---|---|
| Runtime | Node.js 20+ |
| Package manager | pnpm (workspaces) |
| Backend framework | Express 5 |
| ORM | Drizzle ORM (`drizzle-orm/node-postgres`) |

| Auth | Google OAuth 2.0 (manual fetch, not Passport) + JWT |

## Database Schema

All tables live in `lib/db/src/schema/`.

| Table | Key columns |
|---|---|
| `users` | `id`, `email` (unique), `full_name`, `graduation_year`, notification prefs (6 booleans), privacy prefs (3 booleans) |
| `clubs` | `id`, `name` (unique), `description`, `type`, `initial`, `default_day`, `default_location`, `chat_link`, `profile_photo`, `creator_user_id` FK |
| `events` | `id`, `club_id` FK (cascade delete), `title`, `event_date` (text), `event_time` (text), `location`, `description` |
| `enrollments` | `id`, `user_id` FK, `club_id` FK, unique(user_id, club_id) |
| `club_leaders` | `id`, `club_id` FK (cascade delete), `user_id` FK (set null on delete), `name`, `role`, `email` |

## Environment Variables

All variables live in a single `.env` at the repo root. Both api-server and Vite load from this file.

| Variable | Required | Description | Example / Current value |
|---|---|---|---|
| `DATABASE_URL` | Yes | Supabase PostgreSQL direct connection URL | `postgresql://postgres:pass@db.<ref>.supabase.co:5432/postgres` |
| `JWT_SECRET` | Yes | Secret for signing JWTs | any long random string |
| `GOOGLE_CLIENT_ID` | Yes* | OAuth2 client ID from Google Cloud Console | `123456.apps.googleusercontent.com` |
| `GOOGLE_CLIENT_SECRET` | Yes* | OAuth2 client secret | `GOCSPX-...` |
| `GOOGLE_CALLBACK_URL` | Yes | OAuth2 redirect URI registered in Google Console | `http://localhost:8080/api/auth/google/callback` |
| `ALLOWED_EMAIL_DOMAIN` | Yes | Restrict sign-in to this domain | `athenian.edu` |
| `PORT` | No | API server port (default: 8080) | `8080` |
| `API_PORT` | No | Used by Vite proxy (default: 8080) | `8080` |
| `NODE_ENV` | No | `development` or `production` | `development` |
| `APP_NAME` | No | Server-side app name reference | `Station` |
| `VITE_APP_NAME` | No | App display name in browser (default: ClubHub) | `Station` |
| `VITE_SUPPORT_EMAIL` | No | Support email shown on login page | `support@athenian.edu` |
| `BASE_PATH` | No | Vite base path (default: `/`) | `/` |

*`GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET` can be left empty to test with `/api/auth/dev-login` in development.

## Google OAuth Setup

1. Go to [Google Cloud Console](https://console.cloud.google.com/) → **APIs & Services → Credentials**
2. Create a new **OAuth 2.0 Client ID** (Application type: Web application)
3. Under **Authorized redirect URIs**, add:
   - `http://localhost:8080/api/auth/google/callback` (local dev)
   - `https://yourdomain.com/api/auth/google/callback` (production)
4. Copy **Client ID** → `GOOGLE_CLIENT_ID` in `.env`
5. Copy **Client Secret** → `GOOGLE_CLIENT_SECRET` in `.env`
6. Restart the API server

If `GOOGLE_CLIENT_ID` is empty, the OAuth login button returns `?error=oauth_not_configured`. Use the dev-login workaround below.

## Running Locally

### Prerequisites

- Node.js 20+
- pnpm (`npm install -g pnpm`)

### First-time setup

```bash
# 1. Install all workspace dependencies
pnpm install --ignore-scripts

# 2. Fill in .env at the repo root (DATABASE_URL and GOOGLE_* at minimum)
#    See the Environment Variables table above

# 3. Push the Drizzle schema to Supabase (run from your own terminal — requires IPv6)
pnpm --filter @workspace/db run push

# 4. (Optional) Seed sample data
pnpm --filter @workspace/scripts run seed
```

> **Note on database connectivity:** Supabase direct connections use IPv6
> (`db.<ref>.supabase.co`). Some environments (WSL 1, certain Docker setups)
> cannot reach IPv6. Run `pnpm --filter @workspace/db run push` from a machine
> with IPv6 access (standard macOS terminal works fine).

### Starting the dev servers (two terminals)

```bash
# Terminal 1 — API server (builds then starts on :8080)
pnpm --filter @workspace/api-server run dev

# Terminal 2 — Frontend (Vite on :5173, proxies /api/* to :8080)
pnpm --filter @workspace/clubhub run dev
```

Open `http://localhost:5173` in your browser.

### Dev login (no Google credentials required)

```bash
curl http://localhost:8080/api/auth/dev-login
# Returns: { "token": "<jwt>" }
```

Then open `http://localhost:5173/login?token=<jwt>` to log in as the first seeded user.

### Other useful commands

```bash
# Typecheck everything
pnpm run typecheck

# Run backend tests
pnpm --filter @workspace/api-server run test

# Regenerate API client after openapi.yaml changes
pnpm --filter @workspace/api-spec run codegen
```

## Known Issues & Limitations

- **Google OAuth not configured**: `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET` in `.env` are empty. Use dev-login or fill them in from Google Cloud Console.
- **Photo uploads broken locally**: Club profile photos are stored via a Replit Object Storage sidecar (`/storage/upload`). This endpoint will fail outside Replit until replaced with an S3/GCS/Supabase Storage implementation.
- **No email notifications**: The notification bell shows upcoming events from enrolled clubs, but no emails or push notifications are sent.
- **Single-tenant**: One `ALLOWED_EMAIL_DOMAIN` per deployment.
- **Dev-login endpoint**: `GET /api/auth/dev-login` returns 404 in production (`NODE_ENV=production`). Remove before open-sourcing.
- **No pagination on Leadership Hub**: All led clubs load at once.
